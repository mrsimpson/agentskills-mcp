# Configuring Skills

Skills are tracked in `skills-lock.json` at the root of your project. This file is generated by the CLI when you run `add` and is the source of truth for which skills are installed and where they came from.

## Adding Skills

```bash
npx @codemcp/skills add git-workflow github:anthropics/agent-skills/skills/git-workflow
```

This fetches the skill, installs it, and records it in `skills-lock.json`:

```json
{
  "version": 1,
  "skills": {
    "git-workflow": {
      "source": "anthropics/agent-skills",
      "sourceType": "github",
      "computedHash": "abc123..."
    }
  }
}
```

Each entry records the skill name, where it came from, and a content hash for integrity checks. See [Source Specifiers](/reference/source-specifiers) for all supported source formats.

## Restoring Skills

Commit `skills-lock.json` to your repository. Any team member (or CI agent) runs:

```bash
npx @codemcp/skills install
```

...and gets exactly the same skills, at the same versions, from the same sources.

## Skills Directory

Installed skills land in `.agentskills/skills/`. Add it to `.gitignore`:

```gitignore
.agentskills/skills/
```

Each skill is stored as:

```
.agentskills/skills/
├── git-workflow/
│   └── SKILL.md
└── code-review/
    └── SKILL.md
```

## MCP Server Dependencies

Skills can declare which MCP servers they require. Install with dependency validation:

```bash
npx @codemcp/skills install --agent claude
```

Or auto-install missing MCP servers:

```bash
npx @codemcp/skills install --with-mcp --agent cline
npx @codemcp/skills install --with-mcp --agent opencode  # For OpenCode
```

See [MCP Server Dependencies](/reference/mcp-dependencies) for how to declare these in a skill's frontmatter.

## Programmatic Use

To add and install skills from another Node.js project without using the interactive CLI:

```typescript
import { runAdd, runInstallFromLock } from '@codemcp/skills/api';

// Add a skill (writes to skills-lock.json and installs)
await runAdd(['github:anthropics/agent-skills/skills/git-workflow'], {
  skill: ['git-workflow'],
  yes: true,
});

// Re-install everything declared in an existing skills-lock.json
await runInstallFromLock([]);
```

You can also read and write `skills-lock.json` directly:

```typescript
import { readLocalLock, addSkillToLocalLock } from '@codemcp/skills/api';

const lock = await readLocalLock('/path/to/project');
console.log(Object.keys(lock.skills)); // ['git-workflow', ...]
```
